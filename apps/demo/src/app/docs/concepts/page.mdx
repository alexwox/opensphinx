# Concepts

The OpenSphinx runtime model is intentionally narrow:

```text
SessionState -> generateStep() -> EngineStepResponse -> Step -> SphinxForm -> updated SessionState
```

## Source of truth

`SessionState` and `Step` are the runtime source of truth. The engine uses prior answers to decide what comes next, while the React layer only renders the current step and collects submissions.

## Seed steps and adaptive follow-up

Start with deterministic opening steps. After that, let the engine ask follow-up questions only when they add new signal.

## What OpenSphinx does not do

- It does not own your persistence layer.
- It does not require a single model provider.
- It does not expose a wide root package API.
- It does not replace your product-specific evaluation logic.

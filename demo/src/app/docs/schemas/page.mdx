# Schemas

Use <PackageBadge>opensphinx/schemas</PackageBadge> for the shared types and validation contracts that move between your server and UI.

## Why the schemas matter

- They keep server and client in sync.
- They constrain the package API around explicit contracts.
- They make it easier to validate runtime payloads before generating more steps.

## Import example

```ts
import type {
  AnswerValue,
  EngineStepResponse,
  QuizConfig,
  SessionState,
  Step
} from "opensphinx/schemas";
```
